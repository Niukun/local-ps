## 业务概念模型设计规范

### 1. 概述

#### 1.1 规范定位

业务概念模型用于**与业务人员沟通**，以业务语言描述业务对象（实体）及其关系，不涉及任何技术实现细节。它是数据设计的起点，也是数据资产目录L3（业务对象）的模型化表达。

#### 1.2 适用角色

- 数据架构师（主责）
- 业务架构师（参与）
- 产品经理（参与评审）

#### 1.3 输入

- 业务需求文档
- 第一阶段业务模型设计规范中的“业务实体定义”（可复用）

#### 1.4 输出

- 业务对象清单（含数据分类）
- 概念模型图（简化ER图）
- 概念模型说明文档

### 2. 核心概念

| 概念             | 定义                                           | 示例                         |
| :--------------- | :--------------------------------------------- | :--------------------------- |
| **业务对象**     | 业务领域中的核心事物（人、事、物、地）         | 员工、订单、商品             |
| **数据分类**     | 根据数据架构的分类框架确定                     | 主数据、事务数据、观测数据等 |
| **关系**         | 业务对象之间的业务联系（1:1, 1:N, N:N）        | 一个员工拥有一个名片（1:1）  |
| **属性（可选）** | 在概念模型中可不列出全部属性，仅列关键标识属性 | 员工有员工ID、姓名           |

### 3. 数据分类框架（引用自数据架构规范）

| 分类         | 定义                                     | 示例                 |
| :----------- | :--------------------------------------- | :------------------- |
| **主数据**   | 高业务价值、可跨流程跨系统重复使用的数据 | 客户、员工、产品     |
| **事务数据** | 记录业务事件的数据，有较强时效性         | 订单、发票、支付记录 |
| **观测数据** | 观测对象行为/过程的记录数据              | 系统日志、GPS轨迹    |
| **基础数据** | 有限的允许值范围，用作字典               | 合同类型、国家、币种 |
| **规则数据** | 结构化描述业务规则变量                   | 差旅补助规则、评分卡 |
| **报告数据** | 加工处理后用于决策的数据                 | 销售额报表、KPI      |

### 4. 设计步骤

```mermaid
flowchart LR
    A[从业务需求提取业务对象] --> B[对业务对象进行分类]
    B --> C[确定业务对象间关系]
    C --> D[绘制概念模型图]
    D --> E[业务方评审确认]
```



### 5. 概念模型图规范

- 使用矩形表示业务对象，内部标注“业务对象名称（分类）”。
- 使用连线表示关系，标注关系类型（1:1, 1:N, N:N）。
- 不画主外键，不涉及技术细节。

### 6. 输出模板

#### 6.1 业务对象清单

```mermaid
graph LR
    subgraph 主数据
        EMP[员工（主数据）]
        CARD[数字名片（主数据）]
    end
    subgraph 观测数据
        VISIT[访客记录（观测数据）]
    end
    subgraph 事务数据
        LEAD[线索（事务数据）]
    end
    EMP -- 1:1 拥有 --> CARD
    CARD -- 1:N 产生 --> VISIT
    VISIT -- 1:1 生成 --> LEAD
```



| 业务对象名称 | 业务描述               | 数据分类 | 归属主题域 |
| :----------- | :--------------------- | :------- | :--------- |
| 员工         | 企业雇员               | 主数据   | 人-主数据  |
| 数字名片     | 员工线上展示的名片     | 主数据   | 数字营销   |
| 访客记录     | 访客访问名片的行为轨迹 | 观测数据 | 数字营销   |
| 线索         | 潜在客户意向           | 事务数据 | 销售管理   |

#### 6.2 概念模型说明文档

- 简述业务背景和目标。
- 列出所有业务对象及其分类依据。
- 附概念模型图。

### 7. 与第一阶段业务模型设计规范的关系

| 第一阶段的业务模型设计规范         | 第三阶段的概念模型设计规范   |
| :--------------------------------- | :--------------------------- |
| 包含实体、关系、状态机、规则、事件 | 仅包含实体（业务对象）和关系 |
| 用于驱动架构和Spec设计             | 用于数据治理和逻辑模型前置   |
| 输出更详细（属性、状态等）         | 输出轻量，便于业务方快速理解 |

**建议**：

- 若项目规模小，可直接复用第一阶段业务模型中的“实体定义”+“实体关系”作为概念模型。
- 若项目需要独立数据治理或主数据管理，则应单独编制概念模型文档。
