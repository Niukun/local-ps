# 档案扫描件处理软件 — 研发计划

## 技术栈

| 层面       | 技术选型                                      |
| ---------- | --------------------------------------------- |
| 前端框架   | Vue 2.7 + Vuex 3 + Vue Router 3               |
| Canvas 引擎 | Fabric.js 5.5                                  |
| 裁剪组件   | Cropper.js 1.6                                 |
| 构建工具   | @vue/cli-service 5.0 (Webpack 5)               |
| 样式方案   | SCSS                                          |
| 异步处理   | Web Worker                                     |
| 后端       | 本地 AI 推理服务（单独部署，用于 AI 抠图）      |

## 总体策略

按 **优先级 + 依赖关系** 分 5 个阶段迭代，每个阶段产出可运行、可验证的增量。

```
依赖链:
M012(撤销/恢复) ← 所有处理模块 ← M001(文件管理) → M002(选择导航) → M003(裁剪)/M004(旋转)
                                                                    → M013(导出)
                   M005(纠偏)/M006(去黑孔)/M008(色彩) ← 独立，依赖M012
                   M009(拼接)/M010(分割) ← 依赖M001多文件
                   M011(批量) ← 依赖M001 + M003~M008
                   M007(AI抠图) ← 依赖本地后端
```

## 关键技术决策

1. **色彩调整实时预览**：使用 CSS `filter` 实现实时预览，切换工具时通过 Canvas 固化效果
2. **纠偏算法**：前端使用霍夫变换（纯 JS 实现或 OpenCV.js），检测范围 -45°~45°
3. **去黑孔算法**：基于图片边缘像素检测 + 连通域分析，限定边缘向内 50px
4. **批量处理**：Web Worker 中逐张处理，避免阻塞 UI
5. **AI 抠图**：前端将图片 base64 发送到 `localhost` 后端 HTTP API
6. **PDF 导出**：使用 `jspdf` 库，多页 PDF 每页插入一张图片
